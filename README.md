# 食衡 MealTune

> 记录你吃的，调准下一餐。

MealTune 是一款面向普通健康成年人的轻量饮食、体重与运动管理工具。产品根据用户的身体数据、目标和执行偏好生成初始方案；用户记录食物与体重后，系统计算当天营养进度，并给出下一餐或当日后续饮食建议。

当前阶段：**V1.0 产品方案已完成，正在进入原型与开发阶段。**

## 项目背景

很多用户知道自己需要“少吃一点、吃得健康一点”，但在实际执行中仍会遇到这些问题：

- 不知道每天大约应该摄入多少热量和宏量营养素；
- 记录了食物，却无法快速理解当前营养缺口；
- 不知道下一餐应该优先补什么、减少什么；
- 容易被单日体重波动影响，缺少长期趋势判断；
- 传统健康管理产品功能复杂，首次设置和日常记录成本较高。

MealTune 的目标不是让用户学习复杂的营养知识，而是把“记录食物”转化为可执行的下一步建议。

## 核心产品逻辑

```text
填写身体资料
    ↓
选择用户类型
    ↓
生成初始饮食与运动方案
    ↓
记录食物和体重
    ↓
AI 解析食物输入
    ↓
本地食物数据库匹配
    ↓
用户确认食物、重量和生熟状态
    ↓
代码计算营养摄入与剩余目标
    ↓
AI 根据用户类型生成饮食建议
    ↓
查看体重、热量和蛋白质趋势
    ↓
完成每周与每月复盘
```

## 三类用户策略

MealTune 不使用同一套方案覆盖所有用户，而是根据执行偏好提供三种策略。

| 用户类型 | 适用人群 | 饮食策略 | 运动建议 | AI 建议重点 |
| --- | --- | --- | --- | --- |
| 运动配合减脂 | 愿意规律运动，希望饮食限制相对温和 | 维持热量基础上减少约 10%～15% | 每周 2 次基础力量训练，加 2 次有氧或快走 | 运动前后饮食、蛋白质、碳水和恢复 |
| 饮食优先减脂 | 不希望执行复杂运动计划，主要依靠饮食减脂 | 维持热量基础上减少约 15%～20% | 每周 2～3 次快走，基础力量为可选任务 | 热量空间、低脂蛋白质、蔬菜和执行便利性 |
| 健康饮食管理 | 不以明显减重为首要目标，希望改善饮食结构 | 接近维持热量 | 保持步行、减少久坐，每周 1～2 次基础力量活动 | 饮食均衡、蛋白质、蔬菜、水果和膳食纤维 |

三类用户共用同一套产品结构，但初始参数、首页重点、AI 建议角度和周期复盘标准不同。

## V1 核心功能

### 1. 首次使用与方案生成

用户完成以下基础设置后，系统生成第一版饮食与轻量运动方案：

- 用户类型；
- 年龄、生理性别、身高和当前体重；
- 目标体重；
- 日常活动水平；
- 过敏与忌口；
- 是否存在特殊健康限制。

系统展示每日热量、蛋白质、碳水、脂肪、膳食纤维、蔬菜、水果和饮水目标，并将长期目标拆分为更容易执行的阶段目标。

### 2. 自然语言记录食物

用户可以直接输入：

```text
午餐吃了 200 克熟米饭、150 克煎鸡胸肉和 300 克西兰花。
```

AI 负责提取：

- 餐次；
- 食物名称；
- 重量与单位；
- 生重或熟重；
- 烹饪方式。

解析结果必须由用户确认后才能保存。缺少重量或必要生熟状态时，系统要求用户补充信息。

### 3. 手动记录与本地计算

AI 不可用时，用户仍可通过本地食物数据库手动添加食物。

营养数据由标准食物数据和代码计算完成：

```text
营养摄入量 = 每 100 克营养值 × 实际重量 ÷ 100
```

系统汇总当天的：

- 热量；
- 蛋白质；
- 碳水；
- 脂肪；
- 膳食纤维；
- 蔬菜与水果克数；
- 饮水量。

### 4. AI 饮食决策建议

MealTune 将 AI 的职责拆分为两个部分：

| AI 任务 | 负责内容 | 不负责内容 |
| --- | --- | --- |
| 食物解析 | 理解自然语言并输出结构化食物信息 | 不计算热量和营养事实 |
| 饮食建议 | 根据代码计算结果、用户类型和候选食物生成建议 | 不重新计算营养，不修改用户目标 |

建议生成流程：

1. 代码识别当前最重要的 1～2 个营养问题；
2. 根据过敏、忌口和剩余热量进行硬过滤；
3. 从本地食物库筛选候选食物；
4. 代码计算建议克数；
5. AI 负责将结果组织成易理解、可执行的建议。

这样可以减少大模型编造营养数值或推荐不合适食物的风险。

### 5. 今日、记录、我的

V1 一级导航仅保留三个页面：

| 页面 | 用户主要任务 |
| --- | --- |
| 今日 | 查看体重状态、营养进度、AI 建议和今日运动任务 |
| 记录 | 新增、修改或删除饮食，记录体重和饮水 |
| 我的 | 查看当前方案、长期趋势、周期复盘和数据设置 |

### 6. 趋势与周期复盘

趋势页面重点展示：

- 每日体重与 7 日平均体重；
- 每日热量摄入与目标范围；
- 每日蛋白质摄入与达标天数；
- 最近 7 天和最近 30 天趋势。

产品优先使用多日平均和周期数据判断方案效果，不根据单日体重变化频繁调整目标。

每周和每月复盘最多输出：

- 一个整体结论；
- 一个做得较好的地方；
- 一个需要改善的问题；
- 下一阶段最重要的行动重点。

## 产品亮点

### 规则负责事实，AI 负责理解与表达

营养事实、目标计算、候选筛选和安全校验由数据库与代码完成；AI 只处理自然语言和建议表达。

### 按执行偏好生成不同方案

产品不是简单区分“减脂”和“不减脂”，而是进一步考虑用户愿意依靠运动还是饮食完成目标。

### 本地优先，无需注册

核心资料、饮食、体重和趋势数据默认保存在当前设备，用户无需创建账号即可使用主要功能。

### 趋势优先，避免单日误判

体重判断使用 7 日平均和周期比较，降低水分、饮食时间和测量误差造成的短期波动影响。

### AI 失败不影响核心记录

即使模型接口不可用，用户仍然可以手动记录食物、计算营养、查看历史数据和趋势图。

## 数据与隐私设计

V1 采用本地优先的数据策略：

- `IndexedDB`：保存用户资料、饮食记录、体重记录、营养方案和周期复盘；
- `Local Storage`：保存首次设置状态和少量界面配置；
- AI 食物解析仅发送当前输入文字；
- AI 建议仅发送当前任务需要的结构化营养状态、用户类型和必要限制；
- 不收集姓名、住址、联系人和精确位置。

清除浏览器或应用数据后，本地历史记录可能无法恢复。

## 建议技术方案

| 层级 | 建议方案 |
| --- | --- |
| 产品形态 | 移动端优先 Web App / PWA |
| 前端 | React、Next.js 或同类现代前端框架 |
| 本地数据库 | IndexedDB，可使用 Dexie.js 封装 |
| 图表 | Recharts、Chart.js 或同类折线图组件 |
| 食物数据 | 本地 JSON 或 IndexedDB 种子数据 |
| AI 接口 | 通过后端代理调用 OpenAI 兼容模型接口 |
| 部署 | 静态前端加轻量 Serverless API |

模型密钥不应直接暴露在浏览器客户端。

## 推荐开发顺序

### 阶段 1：数据与计算

- 用户资料结构；
- 食物数据库；
- 初始营养方案计算；
- 每日营养汇总。

完成标准：不接入 AI 也能生成方案并计算食物营养。

### 阶段 2：手动记录闭环

- 首次设置；
- 今日页面；
- 手动添加食物；
- 修改与删除记录；
- 体重与饮水记录。

完成标准：用户可以完整地手动使用产品。

### 阶段 3：AI 能力

- 食物解析 JSON Schema；
- 数据库匹配与用户确认；
- 饮食建议 JSON Schema；
- 模型失败兜底。

完成标准：AI 能力稳定，并且不会影响核心数据计算。

### 阶段 4：趋势与复盘

- 7 日平均体重；
- 体重、热量和蛋白质折线图；
- 周复盘与月复盘。

完成标准：形成长期反馈闭环。

### 阶段 5：体验完善

- 空状态与错误提示；
- 示例饮食；
- 过敏与忌口过滤；
- 数据不足状态；
- 全流程验收测试。

## V1 范围边界

V1 暂不包含：

- 注册、登录、云同步和多设备数据合并；
- 食物拍照识别和条形码扫描；
- 餐馆菜、外卖和复杂混合菜自动估算；
- “一碗、一盘、一份”等模糊份量自动换算；
- Apple Health、Google Fit 和智能手表接入；
- 专业训练计划、动作识别和运动热量返还；
- 医疗诊断、疾病治疗或专业营养处方；
- 社区、会员和商业化系统。

## 核心验收标准

- 新用户可以在数分钟内完成首次设置并获得初始方案；
- AI 不可用时，手动记录仍能完成完整流程；
- AI 可以将多种基础食材解析为固定结构；
- 用户确认前，解析结果不会直接写入记录；
- 保存、修改和删除餐次后，当日汇总会同步更新；
- 过敏食物不会进入推荐候选；
- 三类用户在相同营养状态下会获得不同建议角度；
- 用户可以查看 7 日和 30 日体重、热量与蛋白质趋势；
- 刷新或重新打开应用后，本地数据仍然存在；
- 用户无需登录即可使用全部核心功能。

## 健康与安全说明

MealTune 提供的是基于用户记录、标准食物数据和一般健康规则生成的营养摄入估算与生活方式建议，仅用于普通健康管理参考。

本项目：

- 不计算人体真实吸收率；
- 不承诺固定减重结果；
- 不使用“精准代谢”等误导性表述；
- 不替代医生、营养师或其他专业人员的诊断和治疗；
- 不适用于需要医疗营养干预的特殊人群。
